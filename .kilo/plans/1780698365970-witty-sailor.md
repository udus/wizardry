# Plan: Detailed Pixel Sprites via Data Arrays (Option A)

## Goal
Replace current `fillRect` character drawing with per-pixel sprite data rendered from compact color arrays, giving the wizard, skeleton, and goblin distinct, detailed silhouettes. Preserve all gameplay behavior and collision bounds.

---

## Core Architecture

### Sprite Cache System
Add a reusable renderer that converts a 2D color-mapped array into an offscreen canvas, then blits it with `drawImage`.

```javascript
const spriteCache = {};

function getSprite(key, pixelData, palette, scale) {
  const cacheKey = `${key}_${scale}`;
  if (spriteCache[cacheKey]) return spriteCache[cacheKey];
  const rows = pixelData.length;
  const cols = pixelData[0].length;
  const offscreen = document.createElement('canvas');
  offscreen.width = cols * scale;
  offscreen.height = rows * scale;
  const octx = offscreen.getContext('2d');
  for (let y = 0; y < rows; y++) {
    for (let x = 0; x < cols; x++) {
      const colorIdx = pixelData[y][x];
      if (colorIdx === 0) continue; // transparent
      octx.fillStyle = palette[colorIdx];
      octx.fillRect(x * scale, y * scale, scale, scale);
    }
  }
  spriteCache[cacheKey] = offscreen;
  return offscreen;
}
```

- **Palette**: array of hex strings, index `0` = transparent
- **Scale**: use `scale = 2` for crisp pixel-art look (8×12 array → 16×24 canvas)
- **Facing**: use `ctx.save(); ctx.scale(-1, 1); ctx.drawImage(...); ctx.restore()` for left-facing

## Variable Jump Height

### Goal
Make jump height scale with how long the player holds the jump button, rather than applying a fixed boost while held.

### Current behavior (game.js:555-569)
- On press: `this.vy = this.jumpForce` (instant full jump)
- While held: `this.vy += 180 * dt` up to `maxJumpHold = 0.25s`
- On release: `this.jumpHoldTime = 0` (cut boost immediately)

### Desired behavior
- On press: apply full jump impulse
- While held: continue adding upward velocity (capped so it can't exceed the initial impulse)
- On release: gradually reduce upward velocity toward a minimum float value, giving a "short hop" vs "full jump" feel

### Proposed change
1. Add `this.jumpBoost = 0` and `this.jumpBoostMax = 200` to `Player` constructor
2. In `update(dt)`:
   - On jump press: set `this.jumpBoost = this.jumpBoostMax`
   - While `jumpPressed && this.jumpBoost > 0`: `this.vy -= (this.jumpBoost / this.maxJumpHold) * dt`; decrease `this.jumpBoost` by `dt * this.jumpBoostMax / this.maxJumpHold`
   - On release: set `this.jumpBoost = 0` and `this.vy = Math.min(this.vy, -80)` to cut the jump short
3. Clamp `this.vy` so it never exceeds initial `jumpForce` magnitude while boosting

### Tuning knobs
- `jumpBoostMax`: total extra velocity units available during a hold (affects max height)
- `minShortHopVy`: velocity floor on release (affects short-hop height)
- Keep existing `jumpForce = -420` and gravity `vy += 900 * dt` unchanged

### Files Modified
- `game.js` — Player constructor and `update(dt)` jump logic only
